Project Summary: This repository holds a text-based outfit of the well-known traditional card game, Blackjack, implemented entirely in Python. The game plays in the terminal, with text menus input validation loops and dynamic console output to bring about a fast, responsive interactive user experience.
Core Architecture and Code Design: In the design process, Object-Oriented Programming principles have been addressed mainly so that the game components exist as four separate reusable classes.
The Card Class: This class represents the single playing card. It sets the mode of each card (specifically its suit and rank) and relates the rank to its score value.
The Deck Class: This builds a full deck of fifty-two cards on the fly through nested looping on all suits and ranks. Provided is a shuffle method using random library in Python and a deal method to pop cards from the deck.
The Hand Class: It is responsible for keeping track of the complete card collection held by the player or dealer, enabling cards to be added or stripped during play.
The Chips Class: It keeps records of the player's state of financial affairs: total bankroll, checks on the bet amounts against available bankroll, and filing claims against either busting or winning.
Continuous Game Loop and Automated Logic Sequence: This software is structured to use a continuous game loop to reset its environment for consecutive rounds of Blackjack play. The software also provides input validation functionality that prevents players from placing invalid numeric bets or performing unrecognized action commands such as "hit" or "stay." After a player has completed their play turn, an automated logic sequence is then taken over by the dealer's hand which will force the computer-controlled player to hit on two or more hands until it reaches a total of 17 or more in order to comply with typical casino rules regarding dealer-played hands.
